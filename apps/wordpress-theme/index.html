Simple setup

設定は少なく、機能は多く

入力する場所は絞る。けれど、アクセス解析とAIOチェックは最初から入れておく。

アクセス解析

テーマ内で持つか、外部をつなぐかを先に決める。余計なプラグインを増やしにくい。

  • ページビュー、流入元、滞在をあとから見やすい。
  • ヘッダーへまとめて入れて、触る場所を減らす。
  • 初期設定を複雑にしない。

AIOチェック

何を直せばいいかをチェック式で扱う。細かくしすぎず、必要なところだけONにする。

  • ON/OFFだけで切り替えられる。
  • 難しい文言を減らして、迷う時間を短くする。
  • テンプレは増えても、設定は増やしすぎない。

Install guide

WordPressに入れる手順

迷わないように、管理画面で触る順番を固定している。

1. ZIPをアップロード

外観 → テーマ → 新規追加 → テーマのアップロードから、AIO Starter ZIPを選ぶ。

2. AIO Starterを有効化

テーマを有効化したあと、外観 → AIO Starter で設定画面を開く。

3. 色と解析を設定

メインカラー、文字色、GA4、Search Console、内部解析、llms.txtを同じ画面で設定する。

4. 記事テンプレートを使う

投稿編集画面の「パターン」から、比較、レビュー、案件紹介、AI検索FAQ、ランキング、商品紹介を選ぶ。

5. 解析を見る

外観 → AIO Starter の解析欄で、PV、人気記事、流入元を確認する。

6. llms.txtを確認

サイトURLの末尾に /llms.txt を付けて、AIクローラー向けの案内が出るか確認する。

Why it wins

無料テーマと正面勝負しない

Cocoon、Lightning、Astra、GeneratePress、Kadenceは軽さやカスタマイズが強い。AIO Starterは、そこではなく「記事の型」と「書く前の設計」で差別化する。

記事テンプレート7種類

比較、レビュー、案件紹介、AI検索FAQ、ランキング、商品紹介、AIO基本型を最初から入れる。

AIO記事設計メモ

投稿画面で、読者、検索意図、結論、根拠、注意点、CTAを先に固定できる。

軽いまま足す

外部APIや重い管理JSは使わない。投稿画面のメモ、ショートコード、JSON-LD中心で増やす。

Plugins

入れていいプラグイン、減らせるプラグイン

全部をテーマに詰め込むのではなく、テーマで代替できるものと、専門プラグインに任せるものを分ける。

入れていい

バックアップ、セキュリティ、問い合わせフォーム、EC/決済、画像最適化、スパム対策。サイト全体や外部連携に関わるものは専用プラグインでよい。

減らせる

FAQ構造化データ、目次、GA4タグ挿入、Search Console認証、llms.txt、簡単なアクセス解析はテーマ側に入れている。プラグインを増やしすぎない設計。

重複注意

SEO、キャッシュ、解析は重ねるほど良いわけではない。同じ役割のプラグインを複数入れない。
