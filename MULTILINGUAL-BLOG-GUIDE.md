# 多言語ブログ運用ガイド

## 新しい翻訳記事を公開する

1. 管理画面で通常どおり記事を書く。
2. 「記事設定」を開き、「記事の言語」で `日本語` または `English` を選ぶ。
3. 翻訳元・翻訳先の記事がすでに公開済みなら、そのURL末尾のスラッグを「対応する翻訳記事のスラッグ」に入れる。
4. 公開する。

英語記事は自動で `/en/blog/{slug}/`、日本語記事は `/blog/{slug}/` に公開される。

## 対応する翻訳記事のスラッグ

たとえば対応記事のURLが次の場合:

`https://yohelab.com/blog/2026-06-10-example-article/`

入力する値:

`2026-06-10-example-article`

片方の記事に入力すれば、もう片方も自動検出される。両方に入力しておくと、記事削除や移行時にも関係が明確になる。

## 自動で行われるSEO対応

- 言語別URL
- 自己参照canonical
- 日本語・英語・x-defaultのhreflang
- Open Graph locale
- 言語別のHTML `lang`
- BlogPosting JSON-LDの`inLanguage`
- 言語別パンくず
- 動的サイトマップへの言語別URLとalternate登録
- IndexNowへの言語別URL通知

## 注意

- 商品仕様、価格、法務、購入条件は日本語の原文を正とする。
- 翻訳記事は原文と内容・更新日を揃える。
- 対応記事が未公開の間は、翻訳スラッグを空欄にしても公開できる。
