## 1. 致命傷（売上に直接効くもの・優先度★★★）

### 1-1. OG画像が壊れてる
- `og:image` = `https://yohelab.com/yohelab-icon.png`（128×128）
- `twitter:card` = `summary`（小さい方）
- → Twitter/X、LINE、Facebook、Slack、はてブ、すべてのシェアで「豆粒アイコン」表示
- **やること**: 1200×630のOG画像を作る（文標ロゴ＋「AI検索時代のWordPressテーマ ¥5,500」みたいなコピー入り）→ `og:image` 差し替え＋ `twitter:card` を `summary_large_image` に
- **影響**: SNS経由のCTRが体感1.5〜2倍変わる。やらない理由がない

### 1-2. 特商法表記の穴
- 「販売事業者: 事業者情報（氏名／住所／電話番号）は、請求があった場合に遅滞なく開示いたします」
- **問題1**: 消費者庁ガイドライン（2022改定）では「開示請求の宛先と方法」も明示が必要。今は「問い合わせフォームから請求してね」と書かれてない
- **問題2**: 代表者名すら未記載
- **問題3**: Stripe決済使ってる場合、Stripeのコンプライアンスチェックで「特商法の販売事業者情報が不十分」で警告される可能性
- **やること**:
  - 特商法ページに「販売事業者氏名・住所・電話番号の請求は、`/contact/` の問い合わせフォームに『販売者情報請求』と記載して送付。3営業日以内に開示」と書く
  - 開示を求められたら遅滞なく返答できる態勢を本当に整えておく
  - 代表者名は表記推奨（屋号でもOK）

### 1-3. 連絡先がGmailフリーアドレス
- `yohelab2026@gmail.com` をJSで2分割して隠してる（scraping対策のつもり、効果薄い）
- 商取引の信頼性で減点
- **やること**: `hello@yohelab.com` か `support@yohelab.com` の独自ドメインメールを用意（Cloudflare Email Routing 無料、Gmailに転送可）

### 1-4. 問い合わせフォームが formsubmit.co 依存
- 無料サービス、SLAなし、落ちても気付けない
- スパム対策がハニーポット1個のみ（reCAPTCHA / Cloudflare Turnstile 未導入）
- **やること**: Cloudflare Workers か自前API（既にfunctions/フォルダがある）に置き換え、Turnstile（無料）で防御

### 1-5. sitemap.xml に「準備中」ページが5本混ざってる
- `/apps/research-writer/`（準備中）
- `/lp/research-writer/`（準備中）
- `/products/research-writer-beta/`（準備中）
- `/products/page-review/`（準備中）
- `/products/bunsirube/`（同名で重複、これも未完？）
- Google検索でこれを踏んだ人は「準備中」を見て即離脱＝直帰率＆ブランド毀損
- **やること**: sitemap.xml から削除＋ HTML に `<meta name="robots" content="noindex,nofollow">` 入れる。完成見込みがないなら 410 Gone か削除

### 1-6. ブログ記事が薄すぎる
- 9本のうち6本が **400〜600字** しかない
- 検索エンジンもAI Overviewも、薄い記事は引用しない（情報密度で負ける）
- タイトル設計は良い（「無料テーマと文標の違い」「比較記事の書き方」等）→ 中身肉付けすれば化ける
- **やること**: 1本2,500〜4,000字へ書き直し。見出しH2×4〜6、表・FAQ・出典つき。文標自体の機能で「FAQ・比較表・出典・短い答え」を入れろと言ってる手前、自分のブログがやってない

## 2. 売れるようにするための提言（優先度★★）

### 2-1. ヒーローH1の差し替え
- 今: 「AI検索時代の記事構造を、WordPressで迷わず作る。」
- 問題: 抽象。「迷わず作る」だけだと「結局なに？」
- 案: 「**比較記事もレビューも、7種類のテンプレから書き始められる WordPress テーマ**」みたいに**具体名詞**を入れる
- ベネフィット型にするなら: 「**書く前に止まる時間を、テンプレ7種で消す**」など

### 2-2. ベータ価格の売り方を変える
- 今の表現: 「ベータ ¥5,500（税込）」「正式版では¥8,800へ改定予定」を**散らして**書いてる
- 案: ヒーロー脇に
  ```
  ベータ価格 ¥5,500（税込）
  正式版 ¥8,800
  → 今だけ ¥3,300 OFF（37%）
  ベータ購入者は正式版アップデート無料
  ```
  を**箱で囲んで一発表示**。FOMO（取り逃したくない）を発動

### 2-3. 30日返金保証を主役にする
- 今: hero-mini の pill 群に紛れて1枚
- 案: CTAボタンの**真下**に **「30日返金保証つき。理由を問わず全額返金。Stripe経由で5〜10営業日。」** を**赤帯または緑バッジ**で大きく
- 「無料体験」と同等の心理的ハードル下げになる

### 2-4. 社会的証明を作る
- 現状: 「作ってない声は載せない」は誠実、でも購入率に効かない
- アクション3つ：
  1. **無料モニター5〜10人**に配って、許可取れたら**スクショ付きレビュー**を集める（Xで募集すれば集まる）
  2. **自分の運営サイト**（yohelab.com 自体・別のアフィリブログ等）が文標で動いてることを明示
  3. **販売数カウンター**（「今月◯件販売」「累計◯件」）。1件でも入ったらカウント開始

### 2-5. 競合比較表に実名を入れる
- 今: 「無料テーマでやる場合 / 高機能テーマでやる場合 / 文標」← 抽象
- 案: 「**Cocoon（無料）/ SWELL（¥17,600）/ AFFINGER6（¥14,800）/ 文標（¥5,500）**」で価格・記事テンプレ有無・FAQブロック・買い切り or 否を表で見せる
- GPLテーマなので名称言及は法的に問題なし。「dis る」ではなく「住み分け」表現で書けばOK

### 2-6. 「向いている人／向いていない人」を判定フロー化
- 今: カード3枚で文章
- 案: 簡易フローチャート
  ```
  Q1: WordPressでブログを書く予定？
  → No → 向きません
  → Yes → Q2: 比較記事 or レビュー記事を書く予定？
    → No → Cocoon等の無料テーマでもOK
    → Yes → Q3: 月額より買い切りがいい？
      → No → SWELL等の総合テーマがおすすめ
      → Yes → ✅ 文標が向いてる
  ```
- 「自分用」と気付かせるのが最も強いCTA

### 2-7. 「ベータ」という言葉の扱い
- 今: 「ベータ」を6回くらいページ内で連呼
- 心理的影響: 「ベータ＝未完成＝バグありそう」と読まれる
- 案: 「**先行提供版**」「**Early Access**」のような表現に置き換え、自動更新で機能追加されることを強調

### 2-8. CTAボタン文言のA/B案
- 今: 「¥5,500（税込）で購入する」「購入してZIPを受け取る」
- A: 「**今すぐ文標を試す（30日返金保証つき）**」
- B: 「**¥5,500で文標を入手する →**」
- C: 「**¥3,300 OFF で文標を始める →**」
- 価格訴求かベネフィット訴求かで分けてテスト

### 2-9. 内部リンク強化
- 今: ブログ→LP はある。**LP→ブログがない**
- 案: LP の「比較表カード」セクションの下に「**▶ 比較記事の書き方を読む**」、FAQ機能の下に「**▶ AI検索時代のFAQの作り方を読む**」など、関連ブログへの導線を貼る
- ついでにブログ記事を太らせるとSEO面でも回遊面でも効く

### 2-10. 最終CTAをFOMO化
- 今: 「見た目だけじゃなく、記事の作り方まで整える。」← また抽象に戻る
- 案: 「**ベータ価格は◯◯時間で終了予定。¥5,500（税込）で文標を試す →**」
  - 終了時刻を本当に決めると焦りを作れる（決めない自由もあるけど）

## 3. デザイン・UX指摘（優先度★★）

- **ヒーローの装飾オブジェクトが多すぎ**: laptop / phone / robot AI badge / floating tag×3 / orbit ← 0.5秒で何のサイトか伝わらない。半分削って、**実物のWordPress管理画面スクショ**にする方が刺さる
- **/games/ の15ミニゲーム**: 商品サイトに無関係、「何屋さん？」感の原因。別ドメインに切り出すか削除を強く推奨
- **affiliateページだけトンマナ崩れ**: 紫グラデ＋オレンジ＋黄色。サイト全体のティール基調と乖離。統一すべき
- **フッターリンクが page によってバラバラ**: home/LP は affiliate あり、contact/legal にはない。順番も統一されてない
- **favicon が32pxのみ**: apple-touch-icon（180px）、manifest 用の 192/512、ホーム画面追加で荒れる
- **アイコン画像のディテール**: 128pxまでしかなく輪郭がぼやけ気味。SVG版を作ると将来楽
- **カラーコントラスト**: var(--home-muted) `#5b6f8f` × 白背景は AAパス、ただ淡いグレー文字×淡いグレー背景の組み合わせが読みにくい箇所あり（特に hero-lead）
- **LP価格セクション（line 1606〜）の trust-card 9枚**: 情報過多で読まれない。3カテゴリ（決済・サポート／環境／製作運営）に圧縮
- **ヒーロー脇のpill「ベータ ¥5,500（税込）」がリンクじゃない**: pill ごとクリックでStripeへ飛ばすと拾える
- **hero-avatar クラスの中身が `✓`**: アバター用クラス名なのにチェックマーク。意図不明、意味的に変

## 4. バグ・古い記述・抜け（優先度★）

- **LP動画の cache buster `?v=blog-demo-20260504`** が1年前のままで止まってる。ブラウザキャッシュ更新が止まる可能性
- **contact フォームの「対象」と「問い合わせ内容」が意味重複**: 1つに統合できる（DRY）
- **/affiliate/dashboard/ が robots.txt で除外されてない**: ログイン後の管理画面なら noindex 必須
- **JSでメアド分割（`['yohelab2026','gmail.com']`）**: JS実行されれば普通に拾えるのでスクレイピング対策の効果薄
- **LP h1 が30文字超え＆SEOキーワード不足**: 「比較記事」「アフィリエイト」「軽い」「買い切り」のうち「軽い」しか入ってない
- **Lighthouse 100/100/100/100 が「よへラボサイト本体」のスコア**: 文標を入れたWordPressのスコアじゃない。但し書きは入ってるが、文標入りWP環境のスコアも別途載せる方が誠実
- **構造化データ ItemList の itemListElement が1件のみ**: 文標しかない＝商品が1個と分かるのは良いが、ItemList で1件だけ並べる意味は薄い。SoftwareApplication単体で十分かも
- **JSON-LD の Organization に address / sameAs（SNS）が無い**: 信頼性とリッチリザルトに効く
- **WebSite の potentialAction (SearchAction)** が無い: サイト内検索のリッチスニペット未対応（サイト内検索ないなら不要）
- **Microsoft Clarity / Hotjar 未導入**: ヒートマップ・スクロール率取れないと改善判断ができない（Clarityは無料）
- **`<link rel="manifest">` がない**: PWA対応してないなら不要、するなら追加
- **LP の `<meta name="theme-color">` が `#f5fbff`**: モバイルブラウザのアドレスバー色。LPっぽくグリーン系（`#0b8f72`等）の方がブランディング◎
- **/pro/ ディレクトリの中身**: `activate-pending` のみ＝アクティベーション待ち画面用？ 商品紹介ページじゃないなら別パスに移動

## 6. 信頼性・透明性（優先度★★）

- **製作者の顔・名前が出てない**: 「個人開発・個人対応」と言ってるなら、About に名前（屋号OK）＋ X / note へのリンクを入れると信用が一段上がる
- **更新履歴ページがあるのは◎**: ただし「v0.1からv0.3.3まで」と細かすぎるバージョン表記＝**まだまだ未完成感**を与える。「主要アップデート」だけ抜粋する表示モードを用意
- **「自動テスト済み」「AIと人で常に見ています」表現**: 信頼性を増す良い表現。ただ抽象的なので、可能なら **テストの種類・頻度** をもう1段具体的に
- **30日返金保証**: 強い武器。だが「**返金を申請した人の数 / 全体の◯%**」みたいな透明性データを後から出せると最強（販売開始後）

## 10. 抜けてる「あったら売れる」要素

- **デモサイト（実物のWordPress）**: 今は動画と画像のスクリーンショット。実際に触れる管理画面デモサイト（パスワード付きでも）があると即決率上がる
- **「導入1日後の自分」シミュレーション**: 「文標を入れて30分後にこれが書ける」みたいな**時間軸ベネフィット**
- **PDFカタログ／資料DL**: メアド回収できる
- **LINE公式 or メールニュース**: ローンチ告知・割引クーポン配布の手段
- **Twitter（X）の埋め込み**: 自分のX投稿（テーマ開発進捗）を埋め込むと「個人開発の生身感」が出る
- **Beforeの絵 / Afterの絵**: テキストでBefore/Afterはあるが、**画像で記事ページ Before / After** を見せると一発で伝わる
- **「導入実例」コーナー**: 「Aさんの比較記事サイト：文標導入で記事執筆時間が ◯時間 → ◯時間に」（モニター結果で作る）
- **比較動画**: 「Cocoon vs 文標 比較記事を書く時間」みたいな数字で殴る動画
- **無料記事テンプレート**: 「文標を買わなくても使える比較記事テンプレ Notion版」を無料配布 → メアド回収 → 後から文標売り込み
