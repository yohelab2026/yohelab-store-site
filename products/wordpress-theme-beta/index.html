Setup

設定は少なく、機能は多く

アクセス解析

テーマ内で持つか、外部をつなぐかを先に決める。必要な外部サービスだけ選べるようにする。

  • ページビュー、流入元、滞在をあとから見やすい。
  • ヘッダーへまとめて入れて、触る場所を減らす。
  • 初期設定を複雑にしない。

AIOチェック

何を直せばいいかをチェック式で扱う。細かくしすぎず、必要なところだけONにする。

  • ON/OFFだけで切り替えられる。
  • テンプレは増やしても、設定は増やしすぎない。
  • 簡単でわかりやすいのに、使える機能は多い。

Map

何がどこにあるか

ZIP本体

販売ページのダウンロードボタンから取得。外観 → テーマ → 新規追加 → テーマのアップロードで入れる。

設定画面

外観 → AIO Starter。色、GA4、Search Console、内部解析、llms.txtをまとめて設定する。

記事テンプレート

投稿編集画面のパターンから選ぶ。比較、レビュー、案件紹介、AI検索FAQ、ランキング、商品紹介に対応。

ショートコード

READMEに記載。FAQ、まとめ、著者、比較表、引用、目次を記事内に入れられる。

解析

外観 → AIO Starter の解析欄で、PV、人気記事、流入元を確認する。

確認済み環境

WordPress 6.9.4、PHP 8.3.30、wordpress:php8.3-apache、MySQL 8.4 のDocker環境。

Support

サポートと保証の線引き

対応すること

ZIP導入、基本設定、記事テンプレートの使い方、色変更、解析設定、llms.txt、ショートコードの使い方。

別相談にすること

サイト固有のデザイン改造、既存プラグインとの衝突調査、記事制作代行、個別SEO運用。

保証しないこと

検索順位、AI Overview表示、売上、アクセス増加。テーマは軽い土台と設定を提供する。

Free theme gap

無料テーマにない価値

書く型が先にある

比較、レビュー、案件紹介、AI検索FAQ、ランキング、商品紹介、AIO基本型をブロックパターンで選べる。

投稿画面で迷いを減らす

読者、検索意図、結論、根拠、注意点、CTAを残せるAIO記事設計メモを追加。

プラグインを減らす

FAQ、目次、GA4、GSC、llms.txt、軽量解析をテーマ側で持つ。必要な外部プラグインだけ足す。

Plugins

プラグインの判断表

入れていい

バックアップ、セキュリティ、問い合わせフォーム、EC/決済、画像最適化、スパム対策。

基本いらない

FAQ構造化データ専用、目次、GA4タグ挿入、Search Console認証、llms.txt生成、簡単なアクセス解析。

重複注意

SEO、キャッシュ、解析は1系統に絞る。同じ役割を重ねると設定ミスと表示崩れの原因になる。
