Features

文標（ぶんしるべ）に入っているもの全部

JSON-LD

構造化データ自動出力

Article・Person・WebSite・BreadcrumbList・FAQPageをテーマ側で自動生成。プラグイン追加ゼロ。

FAQ

llms.txt + FAQショートコード

/llms.txtを自動生成。一部AIクローラー向けの任意案内で、Google AI Overviews等への表示保証ではありません。FAQショートコードでFAQを設置するとFAQPage JSON-LDも一緒に出力される。

Analytics

内部解析 + GA4・GSC設定

PV・人気記事・流入元をテーマ内で記録。GA4測定IDとGSC認証タグも管理画面から設定できる。

Safety

事故防止チェック

SEO出力の重複、二重キャッシュ、旧ショートコード残り、子テーマ利用状況を管理画面で確認できる。

.htaccess

.htaccess編集

必要な場合だけ直接編集。管理画面に入れない時は復元できないため、保存前バックアップと手元保存を前提に使う。

Fit

AI検索時代の記事構造を、テンプレに最初から入れる

答えが先にある

H1からH3までの並びを崩さず、最初に短い答えを置きやすい順番にする。

FAQが強い

質問と答えを自然に見せて、ページの説明量を無理なく増やす。

根拠と出典を置きやすい

本文に理由、注意点、参考リンクを残し、表示内容と構造化データを合わせやすいベースにする。

Setup

設定は少なく、機能は多く

アクセス解析

テーマ内で持つか、外部をつなぐかを先に決める。必要な外部サービスだけ選べるようにする。

  • ページビュー、流入元、滞在をあとから見やすい。
  • ヘッダーへまとめて入れて、触る場所を減らす。
  • 初期設定を複雑にしない。

記事チェック

何を直せばいいかをチェック式で扱う。細かくしすぎず、必要なところだけONにする。

  • ON/OFFだけで切り替えられる。
  • テンプレは増やしても、設定は増やしすぎない。
  • 簡単でわかりやすいのに、使える機能は多い。

Map

何がどこにあるか

ZIP本体

購入後メールのZIPを取得。外観 → テーマ → 新規追加 → テーマのアップロードで入れる。

設定画面

外観 → 文標。色、GA4、Search Console、内部解析、llms.txtをまとめて設定する。

記事テンプレート

投稿編集画面のパターンから選ぶ。比較、レビュー、案件紹介、FAQ記事、ランキング、商品紹介に対応。

ショートコード

READMEに記載。FAQ、まとめ、著者、比較表カード、CTA、広告リンク、引用、目次を記事内に入れられる。

解析

外観 → 文標 の解析欄で、PV、人気記事、流入元、導線クリックを確認する。

確認済み環境

最低環境はWordPress 6.5以上 / PHP 8.0以上。WordPress 6.8 / PHP 8.3の検証環境で動作確認しています。

Purchase test

購入からシリアル認証まで確認しています

購入メール

Stripe決済Webhookから、購入メール本文に親テーマZIP、子テーマZIP、購入ID、シリアルナンバーが入ることを自動テストしています。

シリアル認証

購入メールのシリアルでライセンス認証が有効になり、不正なシリアルは拒否されることを確認しています。

サーバー確認

主要サーバーは、実機確認できたものだけ「確認済み」と表記します。未確認のものは要件確認中として扱います。

AIと人で継続確認

リンク切れ、表示崩れ、説明不足、使いにくい導線をAIが拾い、人が確認してから安全に更新します。価格・法務・決済・配布ZIPは人の確認を必須にしています。

購入から30日以内であれば、理由を問わず全額返金します。問い合わせフォームから「文標 返金希望」とメールアドレスを送信すればStripe経由で返金されます。返金後はシリアルが無効化され、自動更新は停止します。
