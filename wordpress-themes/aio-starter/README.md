# AIO Starter

AIO Starter は、初心者ブロガー向けの軽量WordPressテーマです。

## 入れ方

1. `aio-starter.zip` をWordPress管理画面からアップロード
2. `外観 > テーマ` で有効化
3. `外観 > AIO Starter` でGA4、Search Console、内部解析、llms.txtを設定
4. メインカラー、文字色、デザインプリセットを必要に応じて変更
5. 投稿で `[aio_toc]`、`[aio_summary]`、`[aio_faq]`、`[aio_author]` を試す

## はじめて使う人向け 5分マニュアル

1. `外観 > AIO Starter` を開く
2. メインカラーと文字色だけ決める
3. アクセス解析は、迷ったらまず「内部アクセス解析ON」だけで始める
4. 投稿編集画面を開く
5. 右サイドバーの `AIO記事設計` で記事タイプを選ぶ
6. `この型の見出しを入れる` を押す
7. `この記事で答えること`、`先に出す答え`、`理由・比較・参考リンク`、`注意点・補足` を短く書く
8. 本文を書いたら `記事完成チェック` を確認する

最初は細かい設定を触らなくて大丈夫です。記事タイプを選び、答えを先に書くことを優先してください。

## 推奨環境

- WordPress 6.5以上
- PHP 8.0以上
- 最新の主要ブラウザ
- 本番利用前にサイトごとの表示、導線、スマホ実機で確認

## サポート範囲

- 対応すること: ZIP導入、基本設定、ショートコードの使い方、GA4/GSC設定欄の使い方
- 別相談にすること: 個別デザイン改造、外部プラグイン衝突調査、既存サイト移行、記事制作代行
- 保証しないこと: 検索順位、AI Overview表示、売上、アクセス増加

## 主な機能

- Article / BlogPosting / Person / WebSite / BreadcrumbList JSON-LD
- `[aio_faq]` と連動する FAQPage JSON-LD
- `/llms.txt` 自動生成
- 軽量内部アクセス解析
- GA4 / Search Console 設定欄
- FAQ、まとめ、著者、比較表、引用、目次ショートコード
- 比較記事、レビュー記事、案件紹介、AI検索向けFAQ、ランキング、商品紹介のブロックパターン
- 投稿編集画面のAIO記事設計メモ（読者、検索意図、結論、根拠、注意点、CTA、チェックリスト）
- 記事タイプ別の見出し挿入ボタン（基本、比較、レビュー、ランキング、FAQ、商品紹介）
- 簡易キャッシュ（コメント受付中の個別記事は除外）
- `comments.php`、`404.php`、`search.php`、`theme.json` あり
- 管理画面からメインカラーと文字色を変更可能

## ショートコード

### まとめブロック（記事冒頭）
```text
[aio_summary title="この記事のポイント"]
・AIO対応とは構造化データを整えること
・FAQを記事に入れるとAIにも人にも意図が伝わりやすい
[/aio_summary]
```

### FAQブロック（FAQPage JSON-LDと連動）
```text
[aio_faq question="AIOとは何ですか？"]
GoogleのAI概要など、検索結果内でAIが回答を要約する流れのことです。
[/aio_faq]
```

### 目次（h2/h3 から自動生成）
```text
[aio_toc title="目次"]
```
※記事の冒頭に置くと、h2/h3 見出しからリンク付き目次を自動生成します。

### 著者情報（E-E-A-T強化）
```text
[aio_author]
[aio_author expertise="SEOコンサルタント" twitter="@username"]
```

### 引用・出典ブロック
```text
[aio_cite source="Google Search Central" url="https://developers.google.com/"]
引用したいテキストを入れます。
[/aio_cite]
```

### 比較表ブロック
```text
[aio_compare]
<table>
  <tr><th>項目</th><th>無料版</th><th>有料版</th></tr>
  <tr><td>テンプレート数</td><td>1種</td><td>3種</td></tr>
</table>
[/aio_compare]
```

## AIO向け記事の書き方

AIO/AI検索時代に読み取りやすい記事構成の例：

```text
[aio_toc]

[aio_summary]
・結論を先に3行でまとめる
・FAQを下部に入れる
・見出し構造をh2→h3の順で整える
[/aio_summary]

## 結論（最初に答えを出す）
本文...

## 詳細説明
本文...

## よくある質問
[aio_faq question="質問1"]回答1[/aio_faq]
[aio_faq question="質問2"]回答2[/aio_faq]

[aio_author expertise="専門分野"]
```

## 最初から入っている記事テンプレート

投稿編集画面のパターンから、次の型を選べます。

- AIO向け基本記事テンプレート
- 比較記事テンプレート
- レビュー記事テンプレート
- 案件紹介記事テンプレート
- AI検索向けFAQ記事テンプレート
- ランキング記事テンプレート
- 商品紹介記事テンプレート

無料テーマとの差別化は、見た目だけではなく「何を書けばいいか」まで最初から用意している点です。結論、根拠、比較、注意点、FAQ、引用欄を置きやすくしています。

## 投稿編集画面のAIO記事設計

投稿・固定ページの編集画面右サイドバーに、軽量な記事設計パネルを追加しています。外部APIや重い解析処理は使いません。

- 記事タイプを選ぶ
- 型に合った見出しを入れる
- この記事で答えることを書く
- 先に出す答えを書く
- 理由・比較・参考リンクを書く
- 注意点・補足を書く
- FAQ、理由、注意点、まとめ、情報の鮮度をチェックする

無料テーマは見た目やレイアウトを整えるものが中心です。AIO Starterは、記事を書く前の設計と、記事タイプ別の見出し作成を投稿画面でできるようにしています。

## プラグイン方針

このテーマは「プラグインを増やさなくてもAIO記事の土台を作れる」ことを目的にしています。すべてをテーマに詰め込むと重くなるため、テーマが持つべきものと、外部プラグインに任せるものを分けています。

### 入れてもよいプラグイン

| 用途 | 推奨方針 | 理由 |
|---|---|---|
| バックアップ | 1つだけ入れる | テーマではサイト全体のバックアップは扱わないため |
| セキュリティ | サーバー側で足りない場合だけ | WAFやログイン保護は環境依存が大きいため |
| お問い合わせフォーム | 必要なサイトだけ | 問い合わせが必要な場合は専用プラグインが安全 |
| EC / 決済 | 販売サイトだけ | WooCommerceなど、決済は専用領域のため |
| 画像最適化 | サーバーにWebP変換がない場合だけ | 画像が多いサイトでは効果が出やすいため |
| スパム対策 | コメントを開ける場合だけ | コメントを使わないサイトでは不要 |

### 基本的に不要、または重複しやすいプラグイン

| 用途 | 判断 | 理由 |
|---|---|---|
| FAQ構造化データ専用 | 原則不要 | `[aio_faq]` とFAQPage JSON-LDをテーマ側で持っているため |
| 目次プラグイン | 原則不要 | `[aio_toc]` をテーマ側で持っているため |
| シンプルなアクセス解析 | 原則不要 | 内部アクセス解析とGA4設定欄があるため |
| GA4タグ挿入だけのプラグイン | 原則不要 | 管理画面からGA4 IDを設定できるため |
| Search Console認証だけのプラグイン | 原則不要 | 認証コード欄をテーマ側で持っているため |
| llms.txt生成プラグイン | 原則不要 | `/llms.txt` をテーマ側で生成するため |
| キャッシュプラグイン | まず不要 | 簡易キャッシュを持つため。大規模サイトではサーバーキャッシュ優先 |
| 多機能SEOプラグイン | 必須ではない | 基本メタ、OGP、JSON-LDはテーマ側で出すため。高度なリダイレクトやサイトマップが必要な場合だけ検討 |

### 注意

プラグインは数だけで良し悪しは決まりません。重複、未更新、不要なCSS/JS、管理されていないプラグインが問題になりやすいです。AIO Starterでは、タグ挿入、FAQ、目次、llms.txt、軽量解析のような小さな用途をテーマ側にまとめ、バックアップや決済のような専門領域は専用プラグインに任せる方針です。
